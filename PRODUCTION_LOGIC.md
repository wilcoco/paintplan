## 3. 스케줄링 로직 (v8.16)

### 3.0 최적화 프레임워크
**제약조건 (Hard Constraints):**
1. 지그교체 예산: 주간 150개, 야간 150개 **절대 초과 불가**
2. 수요 충족: 각 회전 시점까지 해당 수요 반드시 충족

**목적함수 (Objective):**
1. 컬러교환 최소화
2. D+1/D+2 준비 (동일 제약/목적 하에 내일도 최적 운영)

### 3.1 일간 템플릿 안정화
- **하루 동안 동일한 템플릿/순서 유지** (지그교체 최소화)
- 첫 회전에서만 전환 비용 발생, 이후 0
- D0 10회전 → D+1 1회전 전환 시에만 추가 비용

### 3.2 지그 템플릿 결정
1. 해당 일자 전체 수요 기반 지그그룹별 비율 계산
2. 140행어를 수요 비율로 배분 (pcs=2 그룹 우선)
3. 전날 마지막 템플릿과 비교하여 전환 비용 계산
4. 전환 비용이 예산 내면 적용

### 3.3 지그 순서 최적화 (컬러별 묶음)
- **같은 컬러의 지그그룹끼리 연속 배치**
- 이전 날 마지막 컬러 → 오늘 첫 컬러 연결 우선
- 예: SWP그룹(I,B2,B,C) → PLU그룹(H,A) → 기타(D,E,F)
- 회전 내 컬러교환 최소화

### 3.4 예산 관리
- 주간(1-5회전): 150개 예산
- 야간(6-10회전): 150개 예산
- 첫 회전 전환에 대부분 사용, 이후 0으로 유지
- 템플릿 변경이 필요하면 예산 내에서만 허용

## 4. 생산량 배분

### 4.1 D0 + D+1 수요 충족
- D0 각 회전 시점에서 재고 부족이 발생하지 않도록 보정
- **D+1 부족분 선반영**: D0 생산만으로 D+1 전체 수요 커버
- D+1 회전별 부족 예상 시 D0 마지막 회전에 추가 생산
- D+2는 D+1 생산으로 커버

### 4.2 용량 배분
- 지그그룹별 용량 = 행어수 × 2지그 × PCS/지그
- 아이템별 수요 비율로 용량 배분

## 5. 비용 계산

### 5.1 지그교체 비용 (위치 기반) ← 핵심 로직

**원리:** 140개 행어 각 위치에서 이전 회전과 지그그룹이 다르면 교체 공수 발생

**계산 알고리즘:**
```python
def calc_jig_change(prev_template, curr_template, prev_order):
    # 1. 이전 템플릿을 위치 배열로 변환 (순서 유지)
    # 예: {I:35, A:33, H:25} + order=[I,A,H]
    #   → [I,I,...(35개),A,A,...(33개),H,H,...(25개)]

    # 2. 현재 템플릿도 이전 순서 기준으로 위치 배열 변환
    # 이전 순서 유지하여 불필요한 교체 방지

    # 3. 140개 위치별 비교
    changes = 0
    for pos in range(140):
        if prev_positions[pos] != curr_positions[pos]:
            changes += 1
    return changes
```

**예시:**
```
2회전: I:35 → A:33 → H:25 → B2:25 → B:12 → C:10 (순서: [I,A,H,B2,B,C])
3회전: I:35 → A:50 → H:25 → B2:25 → B:5      (순서 유지)

위치별 비교:
- 위치 1-35:   I vs I   = 0 교체
- 위치 36-68:  A vs A   = 0 교체
- 위치 69-85:  H vs A   = 17 교체 (A가 확장되어 H 자리 침범)
- 위치 86-93:  H vs H   = 0 교체
- 위치 94-110: B2 vs H  = 17 교체
- 위치 111-118: B2 vs B2 = 0 교체
- 위치 119-130: B vs B2  = 12 교체
- 위치 131-135: C vs B2  = 5 교체
- 위치 136-140: C vs B   = 5 교체

총 지그교체: 17+17+12+5+5 = 56개
```

**순서 유지 원칙:**
- 이전 회전의 지그그룹 순서를 최대한 유지
- 새로 추가되는 지그그룹은 뒤에 배치
- 제거되는 지그그룹은 순서에서 제외

### 5.2 컬러교환 비용 (지그 순서 기반)
- 컬러 전환 1회 = 3빈행어 = 6개 손실
- **회전 내 교환**: 지그그룹 순서대로 컬러 변경 시 카운트
  - 예: I:SWP → A:PLU → H:PLU = 1회 교환 (SWP→PLU)
- **회전 간 교환**: 이전 회전 마지막 컬러 → 현재 회전 첫 컬러 다르면 카운트
- D+1 부족분 선반영 후 최종 생산량 기준으로 재계산
